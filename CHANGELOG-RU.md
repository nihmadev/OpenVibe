# История изменений (Changelog)

Все ключевые изменения проекта OpenVibe документируются в данном файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/),
и этот проект придерживается [Семантического версионирования](https://semver.org/lang/ru/).

## [1.3.5](https://github.com/nihmadev/OpenVibe/compare/v1.3.4...v1.3.5) — 19.07.2026

Релиз делает запуски агента понятнее и дает ему более безопасный и полезный контекст проекта, а также улучшает редактор, поиск, обновления и кроссплатформенную работу.

### Добавлено

- **Запуски агента и постоянный план**:
  - Добавлен постоянный инструмент `todo` со статусами задач, приоритетами, зависимостями, критериями приемки, доказательствами и checkpoint-данными.
  - Связанные рассуждения и вызовы инструментов объединяются в понятные блоки активности с кратким описанием, временем выполнения и прогрессом.
  - Улучшено отображение рассуждений агента, результатов инструментов, остановленных запусков и ошибок.

- **Работа агента с Git**:
  - Добавлены read-only-инструменты для Git status, branches, refs, log, diff, show, blame и merge-base.
  - В поиск по коду добавлена область поиска: файл, каталог, crate или package из запроса пользователя можно указать явно.
  - Добавлены проверка аргументов и ограничение размера результатов Git-инструментов.

- **Редактор, LSP и поиск**:
  - Добавлены inline-панель Vibe и Monaco LSP-клиент для более полноценной помощи при работе с кодом.
  - Улучшены просмотр Git diff, переходы по редактору и файлам, дерево результатов поиска, фильтрация и навигация с клавиатуры.
  - Добавлены безопасная санитизация Markdown и структурированное отображение дерева файлов.

- **Основа desktop и интерфейса**:
  - Обновлены loading-, welcome-, titlebar-, terminal- и settings-поверхности.
  - Добавлены политика обновлений и сравнение версий для интерактивного и неинтерактивного запуска desktop-updater.
  - Расширено тестовое покрытие путей, запусков агента, todo, поиска, updater и Markdown.

### Исправлено

- Сохраняются корректные сообщения ассистента с вызовами инструментов, а автономное исследование ограничено, чтобы диалоги с провайдерами оставались корректными и сфокусированными.
- Улучшена обработка невалидных аргументов инструментов, отмены, SSE-стриминга и переходов состояний агента/чата.
- Исправлена обработка путей на разных платформах и крайних случаев Git history/diff, включая отсутствующие файлы и scoped paths.
- Небезопасный Markdown и некорректные версии обновлений больше не попадают в интерфейс или updater.
- Обновлены ресурсы локалей; непереведенные строки больше не подменяются некорректным текстом.

### Изменено

- Версия приложения, desktop-пакета, Tauri-манифеста и release-метаданных обновлена до `1.3.5`.

## [1.3.4](https://github.com/nihmadev/OpenVibe/compare/v1.3.3...v1.3.4) — 19.07.2026

Релиз расширяет OpenVibe до полноценной среды разработки: добавляет основу LSP с автоматической установкой инструментов, углубляет работу с Git, обновляет первый запуск и оболочку приложения, повышает надежность подключения к провайдерам и обновляет локализации.

### Добавлено

- **Language Server Protocol (LSP)**:
  - Новый бэкенд-крайт `crates/lsp` с конфигурацией серверов, управлением жизненным циклом процессов, stdin/stdout/stderr, переменными окружения и поиском исполняемых файлов на Windows, macOS и Linux.
  - Готовые профили для Go, TypeScript/JavaScript, Lua, Rust, Python, HTML, CSS/SCSS/Less, JSON, Ruby, C/C++, Java, C# и PHP.
  - Tauri-команды для получения списка серверов и запуска выбранного сервера из интерфейса.
  - Менеджер рантаймов с загрузкой/распаковкой Node.js, Go, Lua, Rust Analyzer, Clangd и JDTLS, а также установкой инструментов через npm, pip, gem, dotnet и Go.
  - Изолированный каталог рантаймов приложения, составление PATH и разрешение абсолютных путей позволяют запускать установленные инструменты без ручной настройки глобального окружения.
  - Исполнитель инструментов агента подключен к общему LSP-менеджеру для языково-зависимых сценариев.

- **Панель Servers и управление LSP**:
  - В Titlebar появился popover с вкладками MCP и LSP.
  - Для каждого LSP-сервера доступны переключатель, состояния установки/работы/остановки/ошибки и индикатор прогресса установки.
  - Общий индикатор MCP/LSP в заголовке отражает запуск, успешное выполнение, остановку и ошибки серверов.

- **Git diff и история**:
  - В редакторе появился виртуальный документ `git-diff:` на базе Monaco Diff Editor.
  - Поддержаны сравнения рабочей копии (`HEAD` ↔ working tree), staged-изменений (`HEAD` ↔ index) и коммита (родительский коммит ↔ выбранный коммит).
  - Добавлена загрузка содержимого файла из Git-refs, включая корректное отображение новых и удаленных файлов и абсолютных путей.
  - Git-панель получила более ясные секции staged/unstaged, переключение плоского и древовидного вида, фильтр, обновление, массовые stage/unstage-действия, открытие diff по файлу, создание/переключение веток и выполнение commit.
  - Граф коммитов обновлен: swimlane-разметка, метки веток и remote, раскрываемые деревья измененных файлов, детальная выборка коммита и всплывающие подсказки, не выходящие за границы окна.

- **Новый onboarding и состояния загрузки**:
  - Минимальный экран первого запуска заменен пошаговым welcome-flow с настройкой имени, языка, темы, цветовой схемы, стиля/скорости анимаций, поведения модели, shell терминала, отображения дерева файлов, звуков, ligatures/размера шрифта редактора и шрифта интерфейса.
  - Имя пользователя автоматически берется из ОС, настройки сохраняются, а шрифт интерфейса применяется сразу.
  - Добавлены переиспользуемые shimmer skeleton-компоненты для загрузки приложения и дерева файлов.
  - Error boundary изолирует ошибки welcome-экрана, чтобы поврежденная настройка или перевод не блокировали запуск.

- **Оболочка, навигация и общие компоненты**:
  - Переработаны layout и Titlebar, активные состояния, ресайз областей чата/поиска/Git/терминала и контекстные меню скрытия/возврата кнопок.
  - Улучшены загрузка и активный элемент File Tree, отступы Search Popup, поведение вкладок терминала и общие Select/Settings-контролы.
  - Расширен набор иконок, уточнены палитры Gruvbox и цвета Monaco Diff Editor.
  - Добавлены тесты утилит путей и UI-поведения, ужесточены глобальные стили и scrollbar-настройки.

- **Надежность агента и сохранение чатов**:
  - Пользовательское сообщение сохраняется до начала генерации, поэтому новый запрос не теряется при ошибке или прерывании.
  - Ошибки инструментов передаются модели как диагностические подсказки с единым маркером, а UI может не показывать сырой технический вывод; невалидный JSON аргументов обрабатывается так же.
  - Системные инструкции агента теперь по умолчанию используют компактные Markdown-блоки `tree` для структуры проекта и файлов.
  - Добавлены Git-снимки файлов для состояний `WORKING`, `INDEX` и commit refs.

- **Прокси провайдеров и сетевой слой**:
  - Добавлен allowlist upstream-хостов, включая Azure OpenAI и AWS, перед проксированием пользовательского base URL.
  - Исправлены заголовки авторизации и версии Anthropic, а также обработка заголовков Google для chat и списка моделей.
  - Увеличены пулы keep-alive-соединений, чтение warm-provider стало безопасным при конкуренции, SSE-буферы переиспользуются через pool.
  - Настройка regional proxy теперь одинаково учитывается для запросов агента и получения моделей; прокси можно отключить.

- **Локализация и обслуживание проекта**:
  - Обновлены строки onboarding, LSP, Servers, Git, loading и общих контролов во всех поддерживаемых локалях, отдельно переработаны китайские (简体/繁體) и вьетнамские ресурсы.
  - Добавлены недостающие ресурсы и ключи переводов, файлы локалей нормализованы.
  - Виртуальное окружение переводов исключено из lint, устаревшие скрипты перевода удалены.

### Исправлено

- Ошибка welcome-экрана больше не блокирует запуск приложения.
- Прокси не отправляет запросы на произвольные хосты и единообразно обрабатывает авторизацию разных провайдеров.
- Diff новых и удаленных Git-файлов корректно отображается, даже если одна сторона сравнения отсутствует.
- Ошибки выполнения инструментов и невалидные JSON-аргументы единообразно отображаются агенту и интерфейсу.

### Изменено

- Версия приложения, desktop-пакета, Tauri-манифеста и release-метаданных обновлена до `1.3.4`.

## [1.3.0] — 15.07.2026

### Добавлено

- **Интеграция Model Context Protocol (MCP)**:
  - Полноценный клиентский бэкенд-движок MCP (`crates/mcp`) с поддержкой транспорта JSON-RPC и управления локальными stdio-серверами.
  - Специализированная панель настроек MCP (`McpSettingsPanel.tsx`) для добавления, редактирования, включения/выключения stdio-серверов, управления переменными окружения и аргументами запуска.
  - Индикатор статуса MCP в заголовке окна (Titlebar) с отображением активных серверов, доступных инструментов и текущего состояния подключения.
  - Визуализация выполнения инструментов в чате с ИИ-агентом (`AgentToolView.tsx`) со стримингом аргументов в реальном времени, выводом ответов, задержкой выполнения и статусной индикацией.

- **Быстрый поиск по коду (Search in Code)**:
  - Высокопроизводительная панель поиска по содержимому файлов (`SearchInCode.tsx`) с двумя режимами отображения: виртуализированный список (`@tanstack/react-virtual`) и древовидная структура папок.
  - Поддержка многопоточного обхода файлов (`jwalk`), учет правил `.gitignore`, кэширование результатов, поиск по регулярным выражениям (Regex), учет регистра, поиск целых слов и фильтрация по шаблонам Include/Exclude.
  - Подсветка найденных совпадений в коде и мгновенный переход на нужную строку файла прямо в редакторе Monaco Editor.
  - Поддержка горячих клавиш (`Ctrl+Shift+F`) и интеграция с палитрой команд (`Command Palette`).

- **Умная генерация контекста 2 (SCG2 — Smart Context Generation 2)**:
  - Встроенный движок интеллектуальной сборки контекста (`crates/scg2`) с легким AST-парсингом символов (структуры, функции, классы, интерфейсы, трейты) для языков Rust, TypeScript, JavaScript, Python и Go.
  - Многофакторная система оценки актуальности кода: затухание по времени обращений к файлам, фокус активных вкладок, близость курсора ввода, ошибки редактора (diagnostics) и неотправленные git-изменения (git delta).
  - Автоматический трекер активности редактора (`scg2Tracker.ts`), передающий события через Tauri IPC (`scg2_push_event`) для непрерывного формирования наиболее релевантных контекстных сниппетов в промпт агента.

- **Модуль рабочей области Git (`crates/git`)**:
  - Выделенный Rust-крайт для проверки состояния git-репозитория, определения имени ветки, отслеживания измененных файлов (staged/unstaged), расчета diff и анализа частоты редактирования файлов.

- **Кастомные правила проекта (.viberules)**:
  - Нативная загрузка и внедрение в системный промпт правил проекта из файлов `.viberules`, `AGENTS.md` и `.cursorrules` из корня открытой рабочей области.

- **Новые темы оформления Gruvbox**:
  - Добавлено 3 варианта цветовой темы Gruvbox: **Gruvbox Hard**, **Gruvbox Medium** и **Gruvbox Soft**.

- **Расширенные настройки кастомных AI-провайдеров**:
  - Гибкая настройка собственных провайдеров: возможность загрузки и выбора SVG-иконки, указание URL для получения списка моделей, установка кастомных JSON-заголовков авторизации и тонкая настройка параметров сэмплинга (temperature, top_p, penalty).

- **Оптимизация предзагрузки типов в редакторе**:
  - Вынос логики предзагрузки типов TypeScript в отдельный крайт `crates/editor`, благодаря чему загрузка файл-деклараций `.d.ts` происходит за один пакетный запрос по IPC.
